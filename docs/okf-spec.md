# OKF v0.1 specification

The Open Knowledge Format (OKF) is an open, vendor-neutral specification (v0.1 Draft, published June 2026 by the Google Cloud Data Cloud team) for representing knowledge as a directory of plain Markdown files with YAML frontmatter. It is designed to be authored by humans, generated by agents, and exchanged across tooling without translation.

## Conformance (§9)

A bundle is conformant with OKF v0.1 if:

1. Every non-reserved `.md` file has a parseable YAML frontmatter block.
2. Every frontmatter block contains a non-empty `type` field.
3. Reserved filenames (`index.md`, `log.md`) follow their defined structure when present.

Only `type` is required; `title`, `description`, `resource`, `tags`, and `timestamp` are recommended. Consumers must tolerate missing optional fields, unknown types, unknown frontmatter keys, and broken cross-links.
